  • Engineers must branch the codebase
Single-Node (bash)
  • 47 standalone bash scripts
  • Separate execution path in CI
  • Different tooling than multi-node
  • Hardcoded server launch commands

How-To

Adding a Benchmark Point

Old Way
  • Copy 143-line YAML file
  • Edit 5+ fields
  • Push to srt-slurm branch
  • Update nvidia-master.yaml
  • Update perf-changelog.yaml
New Way
# Add 1 row to jobs.csv:
dsr1-fp4-b200-new,
  slurm_config + dynamo +
  prefill + decode + bench,
  NODES=3,CONC=[512]

# Validate:
python nvidia_submission/generate_or_validate_sflow_csv.py nvidia_submission/dsr1/fp8/b200/multi_node/sglang/ -v

How-To

New Engine Variant? Copy → Rename → Edit

# 1. Copy closest existing config with descriptive new name
cp decode_tep8_stp.yaml decode_tep4_stp.yaml
#            ^^^^ ^^^^ changed

# 2. Edit the parts that distinguish this variant
vi decode_tep4_stp.yaml
# e.g. add export NCCL_P2P_LEVEL="SYS"
# or add --enable-torch-compile flag

# 3. Add CSV row(s) referencing the new file
# 4. Validate
python generate_or_validate_sflow_csv.py --sflow-validate
